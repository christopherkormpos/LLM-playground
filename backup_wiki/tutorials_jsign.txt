====== Ρυθμίσεις Ψηφιακής Υπογραφής ======
Τα παρακάτω βήματα είναι οι ενέργειες που πραγματοποιούμε στο Helpdesk όταν μας ζητηθεί ψηφιακή υπογραφή σε __MOD PC__ και __INT-PC__. 
  - Ακολουθούμε τις οδηγίες μέσα στο **JSign** φάκελο που βρίσκεται στον δικτυακό μας φάκελο. Δηλαδή:
  - Εγκαθιστούμε το **SafeNet**
  - Εγκαθιστούμε και παραμετροποιούμε το **JSign**
  - Εγκαθιστούμε το **AdobeDC** για INT-PC και το **AdobeReader** (το παλιό) για MOD-PC.

Τα παρακάτω βήματα είναι οι ενέργειες που πραγματοποιούμε στο Helpdesk όταν μας ζητηθεί ψηφιακή υπογραφή σε __Pubadmin(TPC)__ PC πριν δώσουμε τον υπολογιστή στα δίκτυα.
  - Κάνω login ως administrator
  - Ανοίγω Windows Explorer και στην γραμμή διεύθυνσης γράφω: ''%%\\vpnca\sw$%%'' (Εάν το κάνω στο helpdesk αυτό το βήμα τότε το Etherner καλώδιο να είνα στο μπριζάκι "thin install" για να έχω πρόσβαση στον φάκελο)
  - Install **SafeNet**: Πηγαίνω στο φάκελο **SafeNetAuthenticationClient_8.1_SP2 > SafeNet Authentication Client 8.1 SP2 > 32x64 Installer** και εγκαθιστώ το **SafeNet** (που βρίσκω μέσα) με όλες τις ρυθμίσεις default.
  - Install **JSignPDF**: Από το ''%%\\vpnca\sw$%%'' τρέχω το **JsignPdf_setup_1.4.4_wjre** και εγκαθιστώ το πρόγραμμα με όλες τις ρυθμίσεις default.
  - Install **AdobeDC Acrobat**.
  - 
Σε αυτό το σημείο ο Η/Υ έχει το πρόγραμμα (SafeNet Auth Client/ Oberthur) που χειρίζεται το USB token του χρήστη και την εφαρμογή (JSignPDF) που υπογράφει ψηφιακά τα pdf. Οι χρήστες διαμοιράζουν τα ψηφιακά υπογεγραμμένα pdf είτε μέσω ηλ. ταχυδρομείου είτε μέσω της εφαρμογής e-Υπογραφή. 

Σε περίπτωση που είναι εγκατάσταση σ'έναν απ'τους υπολογιστές του ΝΣΚ ή για τους καινούργιους τύπους usb sticks, βάζουμε και τους drivers μέσα απ' το Oberthur.zip

Αναλυτικότερα τα βήματα που πρέπει να γίνουν για τις 2 επιλογές:

===== A. Ψηφιακή υπογραφή pdf μέσω της εφαρμογής **JSignPDF**. ===== 

  - Εγκατάσταση drivers για τις συσκευές token (Safenet/Oberthur) – γίνεται αυτόματα από τα windows κατά την εισαγωγή του usb στη θύρα για 1η φορά.(Σε κάποιους υπολογιστές ίσως χρειαστούν δικαιώματα διαχειριστή κι οι drivers να πρέπει να εγκατασταθούν απ'τον δικτυακό μας φάκελο)
  - Εγκατάσταση εφαρμογής JSignPDF.
  - Παραμετροποίηση εφαρμογής JSignPDF.
  * Είσοδος ως χρήστης (modxxx ή το username του χρήστη).
  * Εισαγωγή token USB, αναμονή για εγκατάσταση του από τα windows (συνήθως ανάβει σταθερά το μπλε λαμπάκι του USB όταν ολοκληρωθεί).
  * Άνοιγμα εφαρμογής JSignPDF, πατάω **[1] προχωρημένη προβολή**, **[2] φόρτωση πιστοποιητικών**. Εφόσον έχει εγκατασταθεί σωστά το token θα βγει στο πεδίο ‘*Πιστοποιητικό*’ το όνομα του κατόχου του token.
{{ :tutorials:jsign-1.png?550 | }}
  * Στο δικτυακό δίσκο (`\\vpnca\sw$` / 10.0.0.15\sw$) , υπάρχει το **JSignPDFSettings.txt** αρχείο. Πατάω το **[3] TSA/OCSP/CRL** κουμπί για να παραμετροποιηθεί η εφαρμογή JSignPDF. Tick στο ‘*Χρησιμοποίησε ασφαλή χρονοσήμανση*’. Βάζω το url του server χρονοσήμανσης στο ανάλογο πεδίο και τα νούμερα στο πεδίο TSA πολιτική (copy/paste).
{{ :tutorials:jsign-2.png | }}
  * Βρίσκω ένα pdf για δοκιμή και το βάζω στο **[4]**, στο **[5]** επιλέγω το όνομα του pdf που θα γίνει export από την εφαρμογή. Πατάω το **[6]** για να ρυθμίσω την εμφάνιση της ψηφιακής υπογραφής πάνω στο έγγραφο. Μέσα στο νέο παράθυρο μπορώ να επιλέξω από το preview που θα βγει η εφαρμογή πάνω στο έγγραφο (συνήθως επάνω δεξιά), και να βάλω και την εικόνα του υπουργείου (υπάρχει στο usb stick και στο δικτυακό δίσκο ypetha.png).
  * Πατώντας το ‘*Υπέγραψε το*’ δοκιμάζω αν πετύχει η υπογραφή.
  * Εγκατάσταση του **Acrobat Reader**, αν δεν υπάρχει (καλό είναι να είναι updated).
  * Παραμετροποίηση του Acrobat Reader. Ανοίγω Acrobat reader Edit > Preferences, category: Signatures, κουμπί more στο πεδίο Verification. Τσεκάρω τα **[1] [2] [3]**.

===== Β. Ψηφιακή υπογραφή pdf μέσω της εφαρμογής e-πογραφή =====
Τα παρακάτω βήματα ακολουθούμε για την εφαρμογή e-πογραφή,όλα τα απαραίτητα αρχέια βρίσκονται μέσα στον δικτυακό φάκελο κάτω απ' το JSign.
  - Δημιουργία λογαριασμού του νέου χρήστη (LDAP).
  - Εγκατάσταση κατάλληλων πιστοιποιητικών, επιλέγοντας εμείς που θέλουμε να εγκατασταθούν ξετικάρωντας το κουτάκι για την αυτόματη εγκατάσταση τους.
  - **ΠΡΟΣΟΧΗ! ΤΑ CERTIFICATES ΠΡΕΠΕΙ ΝΑ ΤΑ ΠΕΡΑΣΟΥΜΕ ΣΤΟ ΛΟΓΑΡΙΑΣΜΟ ΤΟΥ ΧΡΗΣΤΗ ΟΧΙ LOCAL ADMIN**
  - Εγκαθιστούμε το dgs01, επιλέγοντας εγκατάσταση στον προσωπικό φάκελο του χρήστη(Personal Folder).
  - Εγκαθιστούμε το modca, επιλέγοντας εγκατάσταση στις αξιόπιστες πηγές(Trusted Root).
  - **(μόνο για δίκτυο MOD)** Εγκατάσταση κατάλληλου πιστοποιητικού στον browser του χρήστη.
  - **(για δίκτυο Pubadmin)** Εγκατάσταση ειδικού εικονιδίου (χρυσό Chrome) στην επιφάνεια εργασίας του χρήστη.

<callout type="info">Η σελίδα για την e-πογραφή είναι η  https://oraweb.smod.mil.gr:4443/apex/f?p=100 </callout>

<callout type="info">Στα δίκτυα MOD και PUBADMIN τα πιστοποιητικά περνούν μέσα από τις πολιτικές, στην περίπτωση που ο Η/Υ βρίσκεται στο σωστό OU.</callout>

====== Συχνά Προβλήματα ======
<accordion collapsed="true">

<panel title="Έλεγχος ψηφιακής υπογραφής από usb τύπου oberthur">
Σημαντική πληροφορία για αυτού του είδους τα usb είναι ότι χρειάζονται κωδικό, ο οποίος αν δεν έχει αλλάξει είναι "9999" για τον χρήστη και "1234" για τον διαχειριστή. Και με τους δύο κωδικούς(αν δεν έχουν αλλάξει) θα πρέπει να λειτουργήσει η ψηφιακή υπογραφή.Επίσης, προτείνουμε στους χρήστες να αλλάξουν κωδικό.
</panel>

<panel title="Αλλαγή κωδικού ψηφιακής υπογραφής από usb τύπου oberthur">
Σημαντική πληροφορία για αυτού του είδους τα usb είναι ότι χρειάζονται κωδικό, ο οποίος αν δεν έχει αλλάξει είναι "9999" για τον χρήστη και "1234" για τον διαχειριστή. Χρησιμοποιώντας το πρόγραμμα AWP Manager συνδεόμαστε(αφού πρώτα πάρουμε και την έγκριση απ' τον προϊστάμενο του ΤΠΕΝΤ) κι αλλάζουμε τον κωδικό του χρήστη.Αν συμφωνήσει αλλάζουμε και τον κωδικό του διαχειριστή(Εώς 9/12/19 δεν έχει αλλαχθεί κωδικός διαχειριστή). Προσοχή, το πρόγραμμα αυτό το απεγκαθιστούμε μετά απ' τον υπολογιστή του χρήστη έτσι ώστε να μην μπορεί να αποκτήσει ο ίδιος πρόσβαση.
</panel>

<panel title="Ξέχασα τον κωδικό μου">
Ενημερώνουμε τα δίκτυα, αλλά για την αλλαγή του κωδικού σε usb της ψηφιακής υπογραφής χρειάζεται ο χρήστης να πάει σε ΚΕΠ. Αυτά είναι τ' αρμόδια για την αλλαγή κωδικού.
</panel>

<panel title="Δεν εμφανίζεται η εικόνα του υπουργείου">
Μπορεί το μέγεθος της ορατής υπογραφής που έχει επιλέξει ο χρήστης να μην επαρκεί για την εμφάνιση ολόκληρης της ψηφιακής υπογραφής, οπότε αν του γράφει επιτυχημένη επιχείρηση τότε του προτείνουμε να αυξήσει το οριοθετημένο ορθογώνιο που ορίζει για την ορατή ψηφιακή υπογραφή. Μπορεί επίσης να μην έχει επιλεχθεί η εμφάνιση της εικόνας του υπουργείου την οποία επιλέγουμε μέσα απ'τις ρυθμίσεις της ψηφιακής υπογραφής.
</panel>

<panel title="Αποτυγχάνει στην τοποθέτηση της χρονοσήμανσης">
Πρέπει να βεβαιωθούμε ότι τα εκχωρημένα στοιχεία μέσα στις ρυθμίσεις TSA είναι σωστά. Για την καινούργια έκδοση του JSign(16.4) ισχύουν ακριβώς οι ίδιες ρυθμίσεις. Αν τα στοιχεία είναι σωστά κι ο υπολογιστής έχει σύνδεση με το Internet τότε φταίει είτε το JSign είτε ο server του ermis. Αν ο χρήστης θέλει επειγόντως να βάλει υπογραφή τότε αναγκαστικά απενεργοποιούμε προσωρινά την ασφαλή χρονοσήμανση αφού βεβαιωθούμε ότι η ώρα του υπολογιστή, δηλαδή η ώρα που θα χρησιμοποιηθεί απ' την ψηφιακή υπογραφή, είναι σωστή.
</panel>

<panel title="Δεν αναγνωρίζει το usb που περιέχει την ψηφιακή υπογραφή">
Αφού βεβαιωθούμε την μάρκα του usb που περιέχει την ψηφιακή υπογραφή, περνάμε τους αντίστοιχους drivers. Για την στιγμή(2019) είναι είτε το SafeNetAuthenticationClient είτε το Oberthur(Προς το παρόν μόνο για την περίπτωση του ΝΣΚ).
</panel>

<panel title="Δεν εμφανίζεται η χρονοσήμανση">
Μπορεί το μέγεθος της ορατής υπογραφής που έχει επιλέξει ο χρήστης να μην επαρκεί για την εμφάνιση ολόκληρης της ψηφιακής υπογραφής, οπότε αν του γράφει επιτυχημένη επιχείρηση τότε του προτείνουμε να αυξήσει το οριοθετημένο ορθογώνιο που ορίζει για την ορατή ψηφιακή υπογραφή. Μπορεί επίσης να μην έχει επιλεχθεί η εμφάνιση της χρονοσήμανσης η οποία πρέπει να επιλεγεί απ' το εμφάνιση ονόματος και κειμένου μέσα στις ρυθμίσεις ψηφιακής υπογραφής.
</panel>

<panel title="Δεν ανοίγει η σελίδα του e-πογραφή/η σελίδα DGSign">
Η σελίδα DGSign είναι για πρόσβαση της e-πογραφής μέσα απ'το δίκτυο pubadmin. Όμως, η e-πογραφή είναι εφαρμογή του εσωτερικού δικτύου mod. Εφόσον, όλα τα βήματα έχουν γίνει σωστά κι υπάρχουν τα πιστοποιητικά που χρειάζονται(το ελέγχουμε επανεγκαθιστώντας τα πιστοποιητικά), αν ο χρήστης βρίσκεται σε TPCMOD τότε δεν μπορεί να ανοίξει άλλη καρτέλα για να συνδεθεί στο ίντερνετ παράλληλα με την καρτέλα της e-πογραφής. Ταυτόχρονα, δεν μπορεί να ανοίξει την e-πογραφή σε καρτέλα ενός ήδη ανοιχτού browser. Αυτό γίνεται για λόγους ασφαλείας
</panel>
