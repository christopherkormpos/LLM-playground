====== Δίκτυο Pubadmin ======
Το δίκτυο Pubadmin προσφέρει στο χρήστη δυνατότητα σύνδεσης στο Internet μέσω του browser Chrome. Η πρόσβαση σε σελίδες είναι πιο ελεύθερη απ’ότι αυτή του δικτύου MOD ωστόσο και στο δίκτυο Pubadmin ορισμένες σελίδες δεν είναι επισκέψιμες λόγω πολιτικών ασφαλείας που το απαγορεύουν.
</jumbotron>

===== Γενικά =====

Κάθε χρήστης διαθέτει έναν **προσωπικό κωδικό** ο οποίος έχει username με την εξής μορφή: “**MODXXX**”, όπου τα XXX συμβολίζουν τον μοναδικό αριθμό που χαρακτηρίζει το συγκεκριμένο pc. Παραδείγματα σωστών username:

username: ''%%MOD078%%''

password: ''%%**********%%''


username: ''%%MOD159%%''

password: ''%%**********%%''


<callout type="warning">**ΠΡΟΣΟΧΗ:** **__Δεν θα πρέπει να υπάρχει ταύτιση__** μεταξύ του username στο δίκτυο Pubadmin και του δικτύου MOD! Είναι τελείως διαφορετικά δίκτυα και δεν θα πρέπει να υπάρχει ουδεμία συσχέτιση.
</callout>

<callout type="info">Μνημονικός Κανόνας:

  * Όταν το Username είναι το ονομ/νυμο του χρήστη, τότε ο υπολογιστής είναι συνδεδεμένος σε δίκτυο MOD.
  * Όταν το Username είναι της μορφής MOD (π.χ. MOD016), τότε το δίκτυο του τερματικού είναι υποχρεωτικά Pubadmin.
</callout>

===== Setup =====

==== Παραμετροποίηση Pubadmin-PC (TPC), TPCGDAEE, TPCGEE κλπ. ====

  - Αν __δεν είναι__ καταχωρημένο το μηχάνημα(στο section Προσωπικοί Υπολογιστές του ΤΠΕΝΤ portal) το καταχωρούμε βρίσκοντας τη mac address και θέτοντας το Domain ως FREEDOS.
  - Στέλνουμε αίτημα NOC(ΤΠΕΝΤ portal) για Στήσιμο PC->Domain=TPCMOD. Σκανάροντας το barcode με τη συσκευή προστίθενται αυτόματα τα απαραίτητα στοιχεία για το στήσιμο.
  - Αν __είναι__ καταχωρημένο το μηχάνημα σε Domain FREEDOS(στο section Προσωπικοί Υπολογιστές του ΤΠΕΝΤ portal) τότε με την ολοκλήρωση του αιτήματος το αλλάζουμε σε TPCMOD και κολλάμε το καινούριο Barcode.
  - Τοποθετούμε το καλώδιο **ETHERNET** στο μπριζάκι που γράφει **“Thin Install”**
  - Για να ξεκινήσει η εγκατάσταση, ελέγχουμε ότι υπάρχει **MONO** ένα **DVD-ROM Drive** και **ένας Σκληρός Δίσκος**. 
    * Σε περίπτωση που υπάρχει 2η συσκευή την αποσυνδέουμε από την Motherboard.
    * Σε περίπτωση που δεν υπάρχει DVD-ROM η εγκατάσταση θα αποτύχει
  - Κάνουμε Enable να Bootαρει από Lan-Network από τα BIOS.
  - Πατάμε **F12** όταν αρχίσει το Boot από LAN
  - Επιλέγουμε **LiteTouch Windows PE(*86)**
  - Επιλέγουμε **ThinPC01** και κάτω δεξιά **NEXT**
  - Βάζουμε σαν **Computer Name** το όνομα που μας έχουν στείλει από τα δίκτυα( π.χ TPCMOD123)
  - Περιμένουμε να ολοκληρωθεί η εγκατάσταση
  - Συνδεόμαστε με ''%%hdesk%%'' | ''%%tp….!@%%'' και εγκαθιστούμε όλες τις ενημερώσεις.
  - Αντιγράφουμε από τον εξωτερικό σκληρό τον φάκελο drivers pack solution στην επιφάνεια εργασίας και τρέχουμε το αρχείο μέσα σε αυτό. Επιλέγουμε video, sound και usb drivers (όχι κάρτας δικτύου)
  - Τρέχουμε τα windows updates από τον σκληρό (ακολουθώντας τις οδηγίες που υπάρχουν στο txt αρχείο).
  - Τρέχουμε το ninite από το F:\ninite\Ninite
  - Βάζουμε το ethernet στο Test Thin και κάνουμε τα windows updates από τον πίνακα ελέγχου (μέχρι περίπου 200).

==== Έλεγχος Pubadmin-PC ( TPC ) ====

  - Αφού έχει τελειώσει η εγκατάσταση βάζουμε το δικτυακό καλώδιο στο πριζάκι TEST THIN και συνδεόμαστε με τα εξής στοιχεία  
    * Username: ''%%test%%'' 
    * Password: ''%%qw….$%%''
  - Ελέγχουμε αν ο χρήστης (test) υπάρχει στον δίσκο ''%%F:\Users\%%''
  - Ελέγχουμε αν υπάρχει σύνδεση στο Internet, πηγαίνοντας σε μία τυχαία διεύθυνση π.χ ( in.gr )
  - Από τα BIOS αλλάζουμε το Boot Priority (First Boot: HDD,SSD) και κάνουμε DISABLE όλες τις υπόλοιπες συσκευές ΕΚΤΟΣ του σκληρού δίσκου
  - Κλειδώνουμε τα BIOS βάζοντας κωδικό: ''%%YETHA1%%'' ( κεφαλαία )


==== Έλεγχος Στα TPCGDAEE στο Emulation Network ====

  - Αφού έχει τελειώσει η εγκατάσταση συνδεόμαστε με τα εξής στοιχεία  
    * Username: hdesk 
    * Password: tpent….@
  - Πηγαίνουμε στις ρυθμίσεις δικτύου και βάζουμε τα παρακάτω: 
    * **IP**: 192.168.65.1xx (187)
    * **Default Gateway**: 192.168.65.1
    * **DNS**: 192.168.20.10
  - Κάνουμε Log Out, τοποθετούμε το μπριζάκη στο **“Gdaee Test”**  και συνδεόμαστε με **test | qw…#$**
  - Ελέγχουμε αν τρέχει και λειτουργεί το Ninite
  - Ελέγχουμε αν ο χρήστης (test) υπάρχει στον δίσκο `F:\Users\`
  - Ελέγχουμε αν υπάρχει σύνδεση στο Internet, πηγαίνοντας σε μία τυχαία διεύθυνση π.χ (in.gr)
  - Αφού όλα είναι εντάξει συνδεόμαστε πάλι με **hdesk | tpent…@**  και επαναφέρουμε τις ρυθμίσεις δικτύου σε δυναμικές.
  - Από τα BIOS αλλάζουμε το Boot Priority (First Boot: HDD,SSD) και κάνουμε DISABLE όλες τις υπόλοιπες συσκευές ΕΚΤΟΣ του σκληρού δίσκου
  - Κλειδώνουμε τα BIOS βάζοντας κωδικό: **YETHA1** ( κεφαλαία )

==== Windows Not Genuine ====

Σε περίπτωση που συναντήσουμε αυτό το σφάλμα σε κάποιο από τα tpc (tpcmod, tpcgdaee κλπ) τότε ακολουθούμε τα παρακάτω βήματα:

  - Τρέχουμε cmd σε administrative mode (**Run as administrator**)
  - Πληκτρολογούμε την εντολή ''%%slmgr /skms 192.168.20.9%%'' και πατάμε Enter. (Θα εμφανίσει μήνυμα επιτυχούς εγκατάστασης στον τομέα)
  - Εκτελούμε αντίστοιχα την εντολή ''%%slmgr /ato%%''

Σε περίπτωση που **__ΔΕΝ__** επιλυθεί το πρόβλημα εκτελούμε τα επόμενα βήματα:

  - Τρέχουμε cmd σε administrative mode (**Run as administrator**)
  - Εκτελούμε την εντολή ''%%slmgr /ipk 73KQT-CD9G6-K7TQG-66MRP-CQ22C%%'' (θα εμφανίσει μήνυμα επιτυχούς εγκατάστασης του κλειδιού)
  - Εκτελούμε την εντολή ''%%slmgr /ato%%''

===== PUBADMIN Wifi με Ασύρματη Κάρτα Δικτύου - Παραμετροποίηση Κάρτας Δικτύου =====
Για να συνδέσουμε έναν υπολογιστή PUBADMIN στο Wifi Pubadmin εκτελούμε τα εξής βήματα:
  * Συνδέουμε την Ασύρματη κάρτα δικτύου στον υπολογιστή
  * Εγκαθιστούμε τον driver απο το CD
  * Καταγραφή MacAddress της ασύρματης κάρτας δικτύου χρησιμοποιώντας cmd και την εντολή getmac
  * Αποστολή στοιχείων στο γραφείο Συστημάτων (γρ. 444 , εσωτερ 599):
    * Α. MacAddress
    * Β. Γραφείο
    * Γ. Ονομασία ΗΥ
  * Ανοίγουμε τις Ρυθμίσεις Δικτύου και επιλέγουμε Manage Wireless
  * Προσθέτουμε νέα σύνδεση
  * Στο όνομα καταχωρούμε PUBADMIN
  * Στις ρυθμίσεις του δικτύου στην ασφάλεια επιλέγουμε WPA2-Enterprise
  * Στο Encryption Type επιλέγουμε AES
  * Τσεκάρουμε το Start this connection automatically
  * Αφού δημιουργηθεί η σύνδεση στις ιδιότητές, στην καρτέλα security επιλέγουμε settings
  * Αποεπιλέγουμε το validate server certificate
  * Στο configure να δούμε ότι η επιλογή login with domain username and password δεν έχει tick.
  * Επιλέγουμε το fast reconnect, πατάμε OK
  * Στα advanced settings της καρτέλας security τσεκάρουμε το specify authentication mode
  * Επιλέγουμε user authentication, save credentials και καταχωρούμε τα credentials που μας έχουν δωθεί απο το NOC
  * Σώνουμε τις ρυθμίσεις και κάνουμε επανακκίνηση του υπολογιστή.

===== ΧΩΡΙΣΜΟΣ ΔΙΣΚΟΥ ΣΕ PARTITION =====

1) Κάνω σύνδεση ως admin

2) Δεξί κλικ στον Υπολογιστή μου -> manage -> storage -> Disk Management

3) Επιλέγω Δίσκο F -> δεξί κλικ -> shrink -> 5000 (to shrink)

4) Ανοίγω Chrome -> minitool partition wizard ΤΗΝ OFFLINE ΕΚΔΟΣΗ δε θα παίξει η άλλη -> τη κατεβάζω στο δίσκο F

5) Ανοίγω το πρόγραμμα -> Δεξί κλικ στο C -> Extend -> στο take space from βάζω το unlocated που έφτιαξα πριν -> οκ -> apply -> yes -> Restart

** Αν γίνει σωστά θα πρέπει να μου ανοίξει το pc στην οθόνη του Minitool
