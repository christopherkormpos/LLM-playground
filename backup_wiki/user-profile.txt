====== User Profile Cannot Be Loaded ======

Αφού ολοκληρωθεί η διαδικασία αύξησης του OS partition είναι πολύ πιθανό να μην μπορούμε να κάνουμε Login στα Windows και να λαμβάνουμε το σφάλμα: "The User Profile Service service failed the logon. User profile cannot be loaded."

Προκειμένου να το λύσουμε πρέπει να ακολουθηθούν τα παρακάτω βήματα:

  - Συνδεόμαστε στα Windows σε **Safe Mode with Networking**. Για να το κάνουμε αυτό πατάμε επανειλημμένως **F8**, αφού τελειώσει το φόρτωμα του BIOS, και επιλέγουμε **Safe Mode with Networking**
  - Κάνουμε login ως **hdesk**
  - Πατάμε ταυτόχρονα τα πλήκτρα **start + R** και στο παράθυρο που ανοίγει γράφουμε **regedit**
  - Θα ανοίξει το Registry Editor. Εκεί ακολουθούμε το path ''%%HKEY_LOCAL_MACHINE/SOFTWARE/Microsoft/Windows NT/CurrentVersion/ProfileList%%''
  - Παρατηρούμε ότι υπάρχουν δύο φάκελοι με μόνη διαφορά την κατάληξη .bak. Μετονομάζουμε τους φακέλους αυτούς ώστε να μεταφέρουμε την κατάληξη αυτή στον άλλο φάκελο (ανταλλάζουμε ονόματα δηλαδή).
  - Επιλέγουμε το φάκελο που ΔΕΝ έχει .bak κατάληξη. Στο δεξιά παράθυρο βλέπουμε τις μεταβλητές που περιέχει. Πρέπει να διορθώσουμε τις:
    * **ProfileImagePath** Αλλάζουμε το path ώστε να δείχνει στο σωστό σημείο (δηλαδή αλλάζουμε το drive letter πχ το ''%%F:\…%%'' το κάνουμε ''%%D:\…%%'') ανάλογα με το πώς είναι το όνομα του δίσκου που έχει τους Users.
    * **State** Το θέτουμε μηδέν
    * **RefCount** Το θέτουμε μηδέν
  - Επαναλαμβάνουμε τη διαδικασία και σε όποιο φάκελο δούμε ότι το path ProfileImagePath δείχνει σε χρήστη και έχει λάθος drive letter. <callout type="danger">ΔΕΝ ΑΛΛΑΖΟΥΜΕ ΤΟΝ ADMINISTRATOR ΧΡΗΣΤΗ ΜΕ ΤΟΝ ΟΠΟΙΟ ΜΠΑΙΝΟΥΝ ΑΠΟ ΤΑ ΔΙΚΤΥΑ (ΘΑ ΕΙΝΑΙ ΣΤΟΝ C:)</callout>
