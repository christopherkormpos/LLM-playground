===== Router TPENT =====

Το router στο γραφείο του προϊσταμένου είναι αυτό που παρέχει πρόσβαση στο ελεύθερο δίκτυο στους FREE υπολογιστές στο γραφείο του Helpdesk. Στο router αυτό, μέσω του interface του στη διεύθυνση 192.168.1.1, έχει απενεργοποιηθεί η λειτουργία DHCP. Αυτό σημαίνει ότι //δεν// μπορεί να δώσει IPs στους υπολογιστές που συνδέονται σε αυτό. Αυτό συμβαίνει γιατί έχει δημιουργηθεί ένας server σε VM ο οποίος παίζει το ρόλο του DHCP server αντί για το router.

Το πλεονέκτημα της χρήσης του server αυτού είναι η δυνατότητα της χρήσης PXE, δηλαδή η δυνατότητα του να κάνει boot ένας υπολογιστής από το δίκτυό μας. Αυτό χρησιμοποιείται για boot, χωρίς να χρειάζεται η παρουσία δίσκου ή άλλου μέσου αποθήκευσης, εργαλείων όπως Lubuntu, Memtest, Gparted αλλά και Clonezilla για το στήσιμο των Free PC μας.

<callout type="danger">**Προσοχή!** Αυτό σημαίνει ότι εάν για κάποιο λόγο κλείσουν τα VMs ή με οποιοδήποτε άλλο τρόπο απενεργοποιηθεί ο DHCP server τότε __θα υπάρχουν προβλήματα με τη σύνδεση στο ίντερνετ__. Σε περίπτωση που δεν είναι δυνατό να επανατεθεί σε λειτουργία ο DHCP server, μπορούμε εύκολα να ενεργοποιήσουμε το DHCP στο router πηγαίνοντας στη διεύθυνση ''192.168.1.1'', κάνοντας log in και έπειτα Local Network -> LAN -> DHCP Server -> On. Εάν σε επόμενο χρόνο αποκατασταθεί η λειτουργία του αρχικού DHCP server, τότε θα πρέπει να κλείσουμε πάλι το DHCP στο router.</callout>

Αυτός είναι ο server που όπως προαναφέρθηκε λειτουργεί ως DHCP server. Έχει όμως και δεύτερη λειτουργία - σερβίρει τα αρχεία που είναι απαραίτητα για να γίνει το boot from network.

=== DHCP server ===

Εκτός από το κλασικό terminal, αλλαγές μπορούν να γίνουν και μέσω ενός εγκατεστημένου web interface (Webmin) στη διεύθυνση ''https://192.168.1.192:10000/''.

{{ :capture.png?nolink |}}

Μέσα από αυτή τη κατηγορία μπορούμε να παραμετροποιήσουμε τη DHCP λειτουργία. Με τις τωρινές ρυθμίσεις ο server κάνει assign IPs στο εύρος 192.168.1.2 έως 192.168.1.254. Ένα lease, δηλαδή μία δοσμένη IP είναι έγκυρη για 18 ώρες (64800 δευτερόλεπτα στον κώδικα).
Με το πέρας των 18 ωρών οι συνδεδεμένοι υπολογιστές ανανεώνουν τις IPs τους για 18 ακόμη ώρες, ενώ οι IP για τους αποσυνδεδεμένους λήγουν, και παίρνουν μια καινούργια στην επόμενη σύνδεση.

Ο χρόνος αυτός έχει επιλεγεί έτσι ώστε να μην λήγουν οι IPs υπερβολικά συχνά, ταυτόχρονα όμως να μην γεμίζουμε με άχρηστες IPs αφού πολλοί υπολογιστές συνδέονται στο δίκτυο κατά τη διάρκεια της μέρας (πχ για να κάνουν boot από το δίκτυο και να στηθούν ως free, ή για να τρέξουμε κάποιο memtest κλπ).

Τα χαρακτηριστικά αυτά μπορούν να αλλάξουν κάνοντας click στο url που φαίνεται κυκλωμένο στην εικόνα.
{{ :webmin2.png?nolink |}}

Στο πεδίο που φαίνεται στην παρακάτω εικόνα είναι οι καταχωρήσεις για τις στατικές IP. Δηλώνοντας τη MAC address κάποιου μηχανήματος είναι δυνατόν ο server μας να δεσμεύσει και να δίνει πάντα την ίδια, σταθερή IP.

{{ :webmin3.png?nolink |}}

<callout type="warning">Αυτό σημαίνει ότι εάν κλείσει το VM και ενεργοποιηθεί το DHCP στο router, είναι πολύ πιθανό με τη λήξη του lease να αλλάξουν IP τα συστήματα αυτά.</callout>

=== PXE Server (Boot from Network) ===

Ο DHCP server έχει παραμετροποιηθεί έτσι ώστε να δίνει το αρχείο ''/tftpboot/pxelinux.0'' ως boot file.

{{ :webmin4.png?nolink |}}

Το αρχείο ''/tftpboot/pxelinux.cfg/default'' είναι αυτό το οποίο περιέχει όλες τις επιλογές για το boot from network.

{{ :webmin5.png?nolink |}}
